## 2024-05-14 - Accessible Custom Toggle Switch Pattern
**Learning:** When implementing custom Tailwind toggle switches using the `.sr-only peer` pattern (like the "Barter Auction" switch), keyboard focus indicators are completely lost by default because the actual input is hidden. Simply adding `peer-focus:ring` isn't enough; it requires `peer-focus-visible:ring-2 peer-focus-visible:ring-primary` on the visual element to restore native-like keyboard navigation cues. Furthermore, the label must encapsulate the input or use `htmlFor`, and any disjointed descriptive text must be linked via `aria-describedby`.
**Action:** Always verify keyboard navigation (Tab key) on custom form controls. Apply `peer-focus-visible` utilities to the visual representations of `.sr-only peer` inputs, ensure `id` and `htmlFor` are correctly mapped, and use `aria-describedby` for auxiliary explanatory text.
