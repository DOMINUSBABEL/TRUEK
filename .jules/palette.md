## 2026-05-13 - Custom Tailwind Toggle Accessibility
**Learning:** When implementing custom toggle switches with Tailwind CSS using the `.sr-only peer` pattern (hiding the actual checkbox and styling a sibling element), the custom visual element inherently lacks keyboard focus indicators, making it invisible to keyboard-only users who are navigating the page. Furthermore, without explicit `id` linking on the label and `aria-labelledby` / `aria-describedby` on the hidden input, screen readers fail to associate disjointed text near the toggle with the action itself.
**Action:** Always ensure the wrapper around the visual control is a `<label htmlFor="id">`, explicitly add keyboard focus indicators to the visual toggle element (e.g., `peer-focus-visible:ring-2 peer-focus-visible:ring-primary`), and associate nearby text to the hidden input using `aria-labelledby` and `aria-describedby`.
