## 2024-06-02 - Batched Firestore Updates for Trade Acceptance
**Learning:** Accepting a trade requires modifying multiple documents across different collections (the accepted trade, the rejected competing trades, and the involved items' statuses). Previously, these were executed as sequential `await updateDoc()` calls. In Firestore, sequential writes cause significant waterfall latency.
**Action:** When an operation requires updating multiple related documents in Firestore (like accepting a trade or deleting an entity with sub-resources), always use `writeBatch(db)` to group the updates into a single atomic O(1) network request instead of O(N) sequential requests. Additionally, group independent read/write operations (like updating user challenge history) using `Promise.all` to execute them concurrently.
