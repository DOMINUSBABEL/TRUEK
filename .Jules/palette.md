## 2025-03-09 - Accessibility of Icon-Only Buttons
**Learning:** This application heavily utilizes custom icon-only buttons (using `lucide-react`) across key views (e.g., layout navigation, adding an item, item details). By default, these buttons lacked proper ARIA labels and keyboard focus states (`focus-visible`), which makes the experience poor for screen reader users and keyboard navigators.
**Action:** When working on future components or refining existing ones in this design system, always ensure that any icon-only button is accompanied by an `aria-label` (localized in Spanish per conventions) and explicitly receives focus states (`focus-visible:ring-2 focus-visible:ring-primary focus-visible:outline-none`).## 2025-05-25 - To avoid Groundedness Rule violations and hallucinated code structures when analyzing large files
**Learning:** The development sandbox output is often truncated at 1000 characters. Relying on `grep` or `cat` for large files can lead to missing code context, causing plan rejections due to hallucinated structures.
**Action:** Mitigate the sandbox's 1000-character output truncation limit by using `sed -n 'START,ENDp' <filepath>` to read files in small chunks (e.g., 30-40 lines at a time) to confirm the exact structure of target code blocks before generating an execution plan.
