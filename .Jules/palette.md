## 2025-03-09 - Accessibility of Icon-Only Buttons
**Learning:** This application heavily utilizes custom icon-only buttons (using `lucide-react`) across key views (e.g., layout navigation, adding an item, item details). By default, these buttons lacked proper ARIA labels and keyboard focus states (`focus-visible`), which makes the experience poor for screen reader users and keyboard navigators.
**Action:** When working on future components or refining existing ones in this design system, always ensure that any icon-only button is accompanied by an `aria-label` (localized in Spanish per conventions) and explicitly receives focus states (`focus-visible:ring-2 focus-visible:ring-primary focus-visible:outline-none`).
## 2025-03-09 - Accessibility of Custom Tailwind Toggles and Disjointed Labels
**Learning:** In Truekio, standard browser checkboxes are frequently restyled as custom "toggles" using Tailwind's `sr-only peer` pattern. However, descriptive text accompanying these toggles is often placed in separate DOM nodes (like an adjacent `<div>` containing an `<h4>` and `<p>`), breaking label association for screen readers. Additionally, hiding the native checkbox removes native keyboard focus indicators.
**Action:** When creating or fixing custom toggles, always assign an explicit `id` to the hidden `<input type="checkbox">` and wrap any related disjointed descriptive text in a `<label htmlFor="id">` to ensure clicking the text also toggles the checkbox and informs assistive technology. Crucially, append `peer-focus:ring-2 peer-focus:ring-primary` (and potentially offset styles) to the adjacent visual element (`<div>`) to guarantee keyboard focus visibility.
