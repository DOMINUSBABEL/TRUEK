## 2024-03-24 - Touch Target Resizing for Block Labels
**Learning:** In Tailwind, when a `<label>` is set to `block` display (either explicitly or by default in some contexts), its clickable area spans the full width of its parent container. If this label is associated with an input via `htmlFor`, clicking anywhere in that full-width empty space will unintentionally focus the input or trigger the element, leading to frustrating misclicks on mobile devices.
**Action:** Always append the utility classes `w-fit cursor-pointer` to `block` level labels. This restricts the clickable touch target strictly to the text's bounding box and provides visual feedback (the pointer cursor) to indicate exactly where the interactive area ends.
