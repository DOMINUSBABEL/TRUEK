## 2025-03-09 - Accessibility of Icon-Only Buttons
**Learning:** This application heavily utilizes custom icon-only buttons (using `lucide-react`) across key views (e.g., layout navigation, adding an item, item details). By default, these buttons lacked proper ARIA labels and keyboard focus states (`focus-visible`), which makes the experience poor for screen reader users and keyboard navigators.
**Action:** When working on future components or refining existing ones in this design system, always ensure that any icon-only button is accompanied by an `aria-label` (localized in Spanish per conventions) and explicitly receives focus states (`focus-visible:ring-2 focus-visible:ring-primary focus-visible:outline-none`).## 2025-04-14 - Accessibility of Form Inputs and Toggles
**Learning:** The AddItem form used `lucide-react` for icons and standard inputs, but was missing programmatic association between inputs and labels (`id` and `htmlFor`), significantly reducing usability for screen readers and reducing the clickable area for all users. The custom toggle switch (`Barter Auction`) relied on the visual `sr-only peer` Tailwind pattern but did not receive focus states when navigating by keyboard.
**Action:** When creating forms, always link labels to their input fields using matching `htmlFor` and `id` attributes. For custom hidden inputs like toggle switches (`sr-only`), ensure that the visual sibling element has explicit focus ring styles (`peer-focus:ring-2 peer-focus:ring-primary`) so keyboard navigators have visual feedback.
