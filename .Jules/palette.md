## 2025-03-05 - Custom Toggle Accessibility (`peer-focus-visible`)
**Learning:** When implementing custom Tailwind toggle switches using the `.sr-only peer` pattern, the visual wrapper element needs keyboard focus indicators, but a typical focus ring (`peer-focus:ring`) isn't enough for screen readers. Placing the `htmlFor` on a wrapping `<label>` around the `<input>` and associating descriptive text with `aria-describedby` provides explicit context, while adding `peer-focus-visible:ring-2 peer-focus-visible:ring-primary` ensures a robust visual focus indicator only when navigating via keyboard.
**Action:** When creating or modifying custom UI components like checkboxes and toggles using `.sr-only peer`, always ensure the hidden input is explicitly associated with its label/description texts using `aria-labelledby`/`aria-describedby` and apply `peer-focus-visible` instead of `peer-focus` for correct keyboard interaction states.
