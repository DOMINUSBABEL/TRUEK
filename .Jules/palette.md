## 2025-03-09 - Accessibility of Icon-Only Buttons
**Learning:** This application heavily utilizes custom icon-only buttons (using `lucide-react`) across key views (e.g., layout navigation, adding an item, item details). By default, these buttons lacked proper ARIA labels and keyboard focus states (`focus-visible`), which makes the experience poor for screen reader users and keyboard navigators.
**Action:** When working on future components or refining existing ones in this design system, always ensure that any icon-only button is accompanied by an `aria-label` (localized in Spanish per conventions) and explicitly receives focus states (`focus-visible:ring-2 focus-visible:ring-primary focus-visible:outline-none`).## 2025-04-11 - Custom Toggle and Form Accessibility Pattern
**Learning:** In this application, forms and custom components (like the 'Barter Auction' toggle in AddItem.tsx) often lack proper label-to-input association. Custom toggles implemented with the `.sr-only peer` pattern need both disjointed text wrapped in a `<label>` and explicit focus rings (`peer-focus:ring-2 peer-focus:ring-primary`) on the visual indicator to be accessible to screen readers and keyboard users. Furthermore, standard inputs must always pair `id` with `htmlFor` on the `<label>`.
**Action:** When creating forms or custom toggles using Tailwind's peer utility, ensure `id`/`htmlFor` associations are made to expand clickable areas, and use `peer-focus` utilities to render visible focus rings for keyboard navigation.
