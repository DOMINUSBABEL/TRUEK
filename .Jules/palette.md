## 2024-05-18 - Tailwind sr-only Toggle Switch Accessibility
**Learning:** When using the `.sr-only peer` pattern for custom toggle switches in Tailwind, applying `peer-focus-visible:ring-2` to the visual element (the sibling `div`) is crucial for keyboard navigation visibility. Screen readers require the hidden input to have an `id`, its wrapper to have a `htmlFor` pointing to that `id`, and any descriptive text to be linked via `aria-describedby`.
**Action:** Always verify keyboard focus states (tabbing) and screen reader element associations when building or reviewing custom `.sr-only peer` toggles.
